# AI 智囊圆桌 Skill Engine 架构升级报告 (2026-07-12)

本报告汇总了 2026 年 7 月 12 日对 AI 智囊圆桌应用进行的技术引擎重大升级。本次重构极大提升了智囊角色的还原度、网络请求稳定性、Token 开销控制以及响应首字时延 (TTFT)。

---

## 1. 核心升级特性汇总

### 1.1 全动态数据配置与 Seeding (Scheme B)
- **改进前**：智囊角色的中文字段（如 tagline、头像和 order）硬编码在 Kotlin 数据库 callback 中，导致增删及维护极度繁重。
- **改进后**：编写了 `extract_skills_metadata.py` 自动化脚本，自动递归拷贝 7 个 GitHub 角色技能文档树，物理过滤图片/视频大文件，并解析 `SKILL.md` 顶部的 YAML Frontmatter（获取 name, description），最终在 assets 下生成统一的元数据源 `skills_config.json`。Kotlin 客户端通过解析此 JSON 文件实现全动态 seeding 和更新。

### 1.2 向量语义路由 (Vector Semantic Routing — 专家先发)
- **设计背景**：原有的发言顺序完全基于硬编码的 `order` 字段，无法根据用户提问内容做出自适应的主题专家选择。
- **技术实现**：
  - 调用 `text-embedding-004` 将角色的 description 特征向量提取为 768 维 Float 并压缩存入 Room 数据库。
  - 用户提问时，通过 `embedContentWithFallback` 同样获取问题特征向量，计算余弦相似度。
  - 按照相似度降序重排发言人席位，使与提问主题最匹配的领域专家优先作答，大幅提高了脑暴发散的质量。
  - 针对国内网络及超时的不确定性，设计了 **3秒超时本地 Mock 特征向量降级算法**，保证离线测试 100% 畅通。

### 1.3 双模型 Context Broker 决策流水线 (Dual-Model Pipeline)
- **设计背景**：角色技能包含丰富的 references/ 和 examples/，全部载入会带来极大的 Token 浪费和思考首字时延。
- **技术实现**：
  - **决策阶段**：使用轻量快速的 Lite 模型 `gemini-3.1-flash-lite-preview`，读取脑暴历史上下文及候选文档清单，输出最相关的 JSON 数组。
  - **加载拼装**：客户端根据决策只加载选中的文件 Markdown 内容并与主 `SKILL.md` 合流。
  - **生成阶段**：将精炼拼合后的 Prompt 送入主力模型 `gemini-3.5-flash`（配合高强度思考 Thinking Config）生成高度保真、切中要害的角色作答。

### 1.4 会话级 API Key 绑定 (Conversation-level Key Binding)
- **设计背景**：请求级的 Key 轮询使每次请求切换 Key，丧失了 Gemini 服务端隐式上下文缓存（Implicit Context Cache）的命中机会。
- **技术实现**：
  - 基于 `SharedPreferences` 为每一个 `sessionId` 持久化绑定一个固定的可用 Key。
  - 在同一会话的多次聊天中始终维持相同的 Key 发起请求，以命中 implicit cache。
  - 当遭遇 HTTP `429` 频控或其它网络连接 Exception 时，自动触发 24 小时熔断并寻找下一个可用 Key 进行**会话动态强换绑**，实现了极速 TTFT 体验与强故障转移能力的完美平衡。

---

## 2. 数据库变更说明
- 数据库 schema 版本由 1 升级至 **3**。
- `Character` 实体新增了 `skillDescriptionVector` 字段用于持久化 768 维特征向量。
- 增加了 `fallbackToDestructiveMigration()`，确保开发期间数据无缝更新。

## 3. UI/UX 改善
- 席位图顶栏并排新增了 **“专家先发”** (向量语义路由) 的控制 Switch。
- 对“自动顺延”与“专家先发”两个 Switch 组件进行了 `0.7f` 等比缩放，满足严苛的 Material 3 移动端窄屏首行对齐美学。
